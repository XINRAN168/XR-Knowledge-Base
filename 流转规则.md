## 一、核心流转闭环（完善版）

### 1.1 完整闭环图示

```
                    ┌─────────────────────────────────────┐
                    │           📝 反馈闭环                 │
                    ↓                                     │
📊情报·机会 → 💡灵感想法 → 🔮须弥推演 → 🚀OPC → 📝经验沉淀
     ↑              ↓              ↓                       │
     │          💀失败归档 ←─────验证失败                   │
     │              ↓                                      │
     │         案例复盘 → 提取教训                          │
     │              ↓                                      │
     └────────── 新机会发现 ────────────────────────────────┘
```

### 1.2 反馈驱动机制

| 反馈路径 | 起点 | 终点 | 触发条件 |
|:---|:---|:---|:---|
| 经验→机会 | 📝经验沉淀 | 📊情报·机会 | 执行经验中发现新机会 |
| 失败→机会 | 💀失败归档 | 📊情报·机会 | 失败案例复盘后更新策略 |
| 失败→教训 | 💀失败归档 | 经验沉淀 | 提取教训归档 |

### 1.3 流转阶段说明

| 阶段 | 目录 | 状态 | 触发条件 | 时限 |
|:---:|:---|:---|:---|:---|
| 1 | 情报·机会 | 原始情报 | 发现市场机会、政策利好 | 3天 |
| 2 | 灵感想法/待分析 | 初步识别 | 从情报中发现灵感点 | - |
| 3 | 灵感想法/已完善 | 分析完成 | 灵感分析整理完毕 | 7天 |
| 4 | 须弥推演/验证中 | 验证中 | 提交方案进行推演验证 | 5天 |
| 5 | 须弥推演/验证通过 | 已验证 | 推演验证成功 | - |
| 6 | OPC/待执行 | 待执行 | 验证通过的方案进入执行 | - |
| 7 | OPC/执行中 | 执行中 | 开始执行任务 | 按计划 |
| 8 | OPC/已完成 | 已完成 | 任务执行完成 | - |
| 9 | 经验沉淀/归档 | 归档 | 最终经验沉淀归档 | 3天 |

### 1.4 失败路径

| 阶段 | 目录 | 触发条件 |
|:---:|:---|:---|
| A | 灵感想法/已完善 → 💀失败归档 | 灵感分析后判定不可行 |
| B | 须弥推演/验证中 → 💀失败归档 | 推演验证失败 |

## 二、独立话题运行规则

以下话题不参与核心流转，独立运行：

### 🇯🇵 日本·生活

```
┌─────────┐
│ 日语学习 │ ←── 持续学习
├─────────┤
│ 工作准备 │ ←── 项目推进
├─────────┤
│ 生活情报 │ ←── 定期归档
└─────────┘
        ↓
   📝经验沉淀
```

**规则**：
- 日语学习：持续积累，不设终点
- 工作准备：按项目周期管理
- 生活情报：每月整理归档一次
- **流转完成**：各模块完成后 → 📝经验沉淀

### 💰 财务投资

```
┌─────────┐
│ 股市盯盘 │ ←── 实时监控
├─────────┤
│ 投资分析 │ ←── 定期复盘
└─────────┘
        ↓
   📝经验沉淀
```

**规则**：
- 股市盯盘：每日开盘/收盘关注
- 投资分析：每周复盘总结
- 重大决策需进入须弥推演验证
- **流转完成**：投资总结 → 📝经验沉淀

### 💼 工作·职场

```
┌─────────┐
│ 待办事项 │ ←── 日常处理
├─────────┤
│ 项目管理 │ ←── 阶段性推进
└─────────┘
        ↓
   📝经验沉淀
```

**规则**：
- 待办事项：当日事当日毕
- 项目管理：按里程碑节点推进
- 重大决策需进入须弥推演验证
- **流转完成**：项目完成 → 📝经验沉淀

### 🤖 特助·秘书

```
┌─────────┐
│ 日常事务 │ ←── 例行执行
└─────────┘
        ↓
   📝经验沉淀
```

**规则**：
- 提醒事项：按时触发
- 日常事务：自动化处理为主
- 复杂任务转交OPC执行
- **流转完成**：任务完成 → 📝经验沉淀

### 🎴 玄机大师

```
┌─────────┐
│ 玄学应用 │ ←── 决策辅助
├─────────┤
│ 案例研究 │ ←── 经验积累
└─────────┘
        ↓
   📝经验沉淀
```

**规则**：
- 玄学应用：作为决策参考，不作为唯一依据
- 案例研究：记录每次应用场景与结果
- 可与须弥推演配合使用
- **流转完成**：案例研究 → 📝经验沉淀

## 三、模块化配置

### 3.1 情报发现触发

```
触发条件：用户提及市场机会、政策利好、行业动态
↓
动作：
  1. 添加 📊 标签
  2. 创建 情报·机会/待挖掘/新条目.md
  3. 分析机会价值
```

### 3.2 灵感激活触发

```
触发条件：情报中发现潜在机会点
↓
动作：
  1. 添加 💡 标签
  2. 移动到 灵感想法/待分析/
  3. 记录灵感来源（情报链接）
```

### 3.3 推演验证触发

```
触发条件：灵感已完善，需要验证可行性
↓
动作：
  1. 添加 🔮 标签
  2. 移动到 须弥推演/验证中/
  3. 启动须弥推演流程
```

### 3.4 验证结果处理

```
验证通过：
  1. 移动到 须弥推演/验证通过/
  2. 添加 🚀 标签
  3. 创建 OPC/待执行/任务.md

验证失败：
  1. 移动到 须弥推演/验证失败/
  2. 添加 💀 标签
  3. 复制到 失败归档/案例库/
  4. 记录失败原因与教训
```

### 3.5 OPC执行触发

```
触发条件：验证通过的方案进入执行
↓
动作：
  1. 添加 🚀 标签
  2. 移动到 OPC/执行中/
  3. 创建执行计划与时间线
  4. 定期更新进度
```

### 3.6 执行完成触发

```
触发条件：任务执行完毕
↓
动作：
  1. 移动到 OPC/已完成/
  2. 添加 📝 标签
  3. 复制到 经验沉淀/归档/
  4. 撰写总结报告
```

## 六、质量检查点

### 每个流转节点必须检查

1. **完整性**：必要信息是否齐全
2. **关联性**：是否关联上游记录
3. **可追溯性**：决策依据是否记录
4. **可执行性**：结论是否清晰可行

### 流转文件必备要素

```markdown
---
title: [标题]
created: [创建时间]
updated: [更新时间]
status: [当前状态]
source: [来源/触发点]
tags: [相关标签]
---

## 七、模块化设计

> V2.0 新增：模块化配置，支持新增话题无需修改核心逻辑

### 7.1 核心配置文件

| 文件 | 位置 | 说明 |
|:---|:---|:---|
| 话题注册表.json | _核心配置/ | 所有话题的注册信息 |
| 流转规则.json | _核心配置/ | 流转关系配置 |
| 模块接口规范.md | _核心配置/ | 新增话题的标准规范 |

### 7.2 模块配置结构

每个话题模块包含：
```
话题名/
├── 模块配置.json    # 模块独立配置
├── 说明.md          # 模块说明
└── [状态目录]/      # 状态子目录
```

### 7.3 新增话题流程

1. 在 `_核心配置/话题注册表.json` 添加配置
2. 创建对应话题文件夹
3. 添加 `模块配置.json` 和 `说明.md`
4. 在 `流转规则.json` 添加流转关系
5. 无需修改其他模块

### 7.4 话题类型

| 类型 | 说明 | 示例 |
|:---|:---|:---|
| 流转入口 | 起点，无上游 | 情报·机会 |
| 流转型 | 参与核心流转 | 须弥推演、OPC |
| 独立型 | 独立运行 | 日本·生活、财务投资 |
| 终态型 | 流转终点 | 经验沉淀 |
