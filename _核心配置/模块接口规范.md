# 模块接口规范

> 新增话题模块的标准规范，确保模块独立性和系统一致性

---

## 一、模块命名规范

### 1.1 目录命名
- 使用中文名称，与话题名称一致
- 目录内可包含 emoji 作为视觉标识
- 示例：`须弥推演/`、`🔮须弥推演/`、`玄机大师/`

### 1.2 文件命名
- 使用 kebab-case（短横线命名）
- 包含状态标识和创建时间
- 示例：`验证中-商业机会分析-20260418.md`

---

## 二、模块结构规范

### 2.1 必须包含的文件

```
话题模块/
├── 模块配置.json      # 必选：模块独立配置
├── 说明.md            # 必选：模块说明文档
└── [状态目录]/        # 可选：根据模块类型定义
    ├── 子目录1/
    └── 子目录2/
```

### 2.2 模块配置.json 格式

```json
{
  "模块信息": {
    "名称": "话题名称",
    "标签": "emoji标签",
    "类型": "流转型|独立型|流转入口"
  },
  "状态定义": {
    "状态1": "状态说明",
    "状态2": "状态说明"
  },
  "流转关系": {
    "上游": ["上游话题列表"],
    "下游": ["下游话题列表"]
  },
  "时限": "处理时限"
}
```

## 三、话题类型定义

### 3.1 流转入口型
- 角色：系统信息入口
- 特点：无上游，有下游
- 示例：情报·机会

### 3.2 流转型
- 角色：参与核心流转
- 特点：有上游、有下游
- 示例：须弥推演、OPC

### 3.3 独立型
- 角色：独立运行模块
- 特点：无固定流转对象
- 示例：日本·生活、财务投资

### 3.4 终态型
- 角色：流转终点
- 特点：有上游、无下游（或仅反馈到入口）
- 示例：经验沉淀

## 四、新增话题流程

### 步骤1：注册话题
在 `_核心配置/话题注册表.json` 添加配置：

```json
"新话题": {
  "标签": "emoji",
  "类型": "流转型",
  "描述": "话题描述",
  "上游": ["上游话题"],
  "下游": ["下游话题"],
  "状态": ["状态1", "状态2"],
  "时限": "处理时限"
}
```

### 步骤2：创建模块文件夹
```bash
mkdir -p "./话题流转/新话题"
```

### 步骤3：添加必要文件
1. 创建 `模块配置.json`
2. 创建 `说明.md`
3. 创建状态子目录

### 步骤4：更新流转规则
在 `_核心配置/流转规则.json` 添加流转配置

### 步骤5：验证完整性
- 确认目录结构正确
- 确认配置文件格式正确
- 确认与其他模块无冲突

## 五、触发规则

### 5.1 触发识别
当用户输入包含以下关键词时，自动识别对应话题：

| 话题 | 触发词 |
|:---|:---|
| 情报·机会 | 机会、情报、政策、市场 |
| 灵感想法 | 灵感、想法、创意 |
| 须弥推演 | 验证、推演、分析可行性 |
| OPC | 执行、行动、干 |
| 失败归档 | 失败、不行、放弃 |
| 经验沉淀 | 总结、归档、沉淀 |

### 5.2 状态转换
| 当前状态 | 转换条件 | 目标状态 | 目标话题 |
|:---|:---|:---|:---|
| 情报·机会 | 发现机会点 | 灵感想法 | 灵感想法 |
| 灵感想法 | 分析完成 | 待完善 | 灵感想法 |
| 灵感想法 | 不可行 | 失败 | 失败归档 |
| 灵感想法 | 需验证 | 验证中 | 须弥推演 |
| 须弥推演 | 验证通过 | 验证通过 | OPC |
| 须弥推演 | 验证失败 | 验证失败 | 失败归档 |
| OPC | 执行完成 | 已完成 | 经验沉淀 |

## 六、模块独立性原则

### 6.1 松耦合
- 每个模块独立运行，不依赖其他模块状态
- 模块间通过标准化接口通信
- 单个模块故障不影响其他模块

### 6.2 可插拔
- 新增模块无需修改核心流转逻辑
- 删除模块只需更新配置
- 模块可独立测试

### 6.3 配置驱动
- 所有流转规则通过配置文件定义
- 运行时读取配置，不硬编码
- 支持动态调整流转关系

## 七、示例：新增一个话题模块

### 需求
新增"读书笔记"话题，收集阅读书籍的思考

### 实现

#### 1. 注册话题（话题注册表.json）
```json
"读书笔记": {
  "标签": "📚",
  "类型": "独立型",
  "描述": "阅读思考、知识沉淀",
  "上游": [],
  "下游": ["经验沉淀"],
  "状态": ["在读", "已读完"],
  "时限": "按书籍周期"
}
```

#### 2. 创建模块（读书笔记/）
```
读书笔记/
├── 模块配置.json
├── 说明.md
├── 在读/
└── 已读完/
```

#### 3. 流转规则（流转规则.json）
```json
"读书笔记": {
  "流转": "已读完 → 经验沉淀",
  "触发词": "读完", "总结"
}
```

### 完成
无需修改任何其他文件，模块即可正常工作。
