# 去伪存真机制 · 详细说明

> YL思维V9.0 | M3推演系统组件
> 触发条件：情报类信息、外部数据、敏感决策

---

## 四维验证法详解

### 维度一：源头可信度（30%）

| 源头类型 | 可信度 |
|:---|:---:|
| 官方发布 | 95% |
| 权威媒体 | 85% |
| 专业机构 | 80% |
| 社交媒体 | 50% |
| 匿名来源 | 20% |

### 维度二：数据真实性（35%）

- 数据是否可交叉验证
- 数据是否有原始出处
- 数据逻辑是否自洽

### 维度三：时效性（20%）

| 时效 | 权重 |
|:---|:---:|
| 24小时内 | 100% |
| 1周内 | 80% |
| 1月内 | 60% |
| 超过1月 | 30% |

### 维度四：完整性（15%）

- 信息要素是否齐全
- 是否存在明显缺失
- 是否需要补充验证

---

## 处理流程

```
输入信息
    ↓
四维评分
    ↓
┌─────────────────────────────────────┐
│ 可信度 ≥ 80%                        │
│   → 直接处理，正常输出              │
├─────────────────────────────────────┤
│ 可信度 60-80%                       │
│   → 标记存疑，补充验证              │
│   → 输出时说明"该信息存疑"          │
├─────────────────────────────────────┤
│ 可信度 < 60%                        │
│   → 丢弃或标记虚假                  │
│   → 告知用户"信息不可靠"            │
└─────────────────────────────────────┘
```

## 案例

### 案例1：股市消息验证

输入："某大V说某股明天要涨"

- 源头可信度：社交媒体大V → 50%
- 数据真实性：无数据支撑 → 40%
- 时效性：即时 → 100%
- 完整性：缺少理由 → 30%

综合：50×0.3 + 40×0.35 + 100×0.2 + 30×0.15 = 51分

结论：可信度 < 60%，标记存疑，告知用户

### 案例2：政策文件验证

输入："教育部官网发布的政策文件"

- 源头可信度：官方发布 → 95%
- 数据真实性：有原文链接 → 95%
- 时效性：当天发布 → 100%
- 完整性：完整文件 → 95%

综合：95×0.3 + 95×0.35 + 100×0.2 + 95×0.15 = 96分

结论：可信度 ≥ 80%，直接处理
