### 2.2 核心风险详解

#### 🟡 R-09: 评分僵化风险（新增）

```
触发场景：
- 主人工作阶段：偏好高效、简洁
- 主人学习阶段：偏好详细、解释
- 主人情绪变化：不同情感权重需求
- 长期使用后：初始权重可能不再适用

问题：五维权重固定，难以适应主人行为模式变化

建议方案：
1. 增加权重微调机制（主人可手动调整）
2. 增加隐式反馈学习（主人忽略/追问行为）
3. 定期回顾权重合理性（如季度）
```

#### 🟡 R-10: 跨层数据不一致（新增）

```
触发场景：
- PowerMax写入成功，但PowerScale复制延迟
- 云存储同步失败，但上层已标记完成
- 多载体同时写入产生冲突

建议方案：
1. 增加跨层一致性检测任务（每小时）
2. 实现版本向量时钟（Vector Clock）
3. 定义不一致场景下的优先级规则
```
