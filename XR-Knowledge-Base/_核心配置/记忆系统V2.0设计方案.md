## 二、记忆价值判断算法

### 2.1 五维评估模型

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         记忆价值评估模型                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   综合评分 = w₁×重要性 + w₂×时效性 + w₃×关联度 + w₄×提及频率 + w₅×情感权重   │
│                                                                              │
│   其中：                                                                     │
│   w₁ = 0.30  (重要性权重)                                                   │
│   w₂ = 0.20  (时效性权重)                                                   │
│   w₃ = 0.20  (关联度权重)                                                   │
│   w₄ = 0.15  (提及频率权重)                                                 │
│   w₅ = 0.15  (情感权重)                                                     │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 各维度详细定义

#### 2.2.1 重要性评分（0-100）

| 类型 | 初始分 | 说明 |
|:---|:---:|:---|
| **主人明确要求"记住"** | 100 | 最高优先级，直接标记为长期记忆 |
| **主人身份/偏好信息** | 95 | 必要记忆区候选 |
| **核心决策/结论** | 80 | 高价值，需长期保留 |
| **任务进度/状态** | 70 | 中等价值，任务完成后可能降级 |
| **一般对话内容** | 50 | 默认值 |
| **临时查询结果** | 30 | 低价值，可能快速过期 |
| **试错/测试内容** | 10 | 最低价值 |

#### 2.2.2 时效性评分（0-100）

| 类型 | 初始分 | 半衰期 |
|:---|:---:|:---|
| **实时行情/数据** | 100 | 5分钟 |
| **今日任务安排** | 90 | 24小时 |
| **本周项目进度** | 70 | 7天 |
| **月度规划** | 50 | 30天 |
| **年度战略** | 30 | 365天 |
| **历史知识/方法论** | 10 | 永久有效 |

**时效衰减公式**：
```
时效性分数 = 初始分 × e^(-λt)
其中 λ = ln(2) / 半衰期
```

#### 2.2.3 关联度评分（0-100）

| 关联情况 | 评分规则 |
|:---|:---|
| 与主人核心目标直接相关 | 100 |
| 与当前项目/任务相关 | 80 |
| 与历史成功案例相关 | 70 |
| 与失败案例相关（警示） | 60 |
| 与主人兴趣相关 | 50 |
| 独立信息 | 30 |

#### 2.2.4 提及频率评分（0-100）

| 提及次数 | 评分规则 |
|:---|:---|
| 主人明确"记住" | 100 |
| 24小时内提及≥3次 | 90 |
| 7天内提及≥2次 | 70 |
| 30天内提及≥1次 | 50 |
| 从未提及但有价值 | 40 |
| 模糊提及 | 20 |

#### 2.2.5 情感权重评分（0-100）

| 情感类型 | 评分规则 |
|:---|:---|
| 主人明确表达重视/期待 | 100 |
| 主人情绪高涨时分享 | 90 |
| 主人困惑时寻求帮助 | 80 |
| 主人平淡讨论 | 50 |
| 主人不耐烦/敷衍 | 20 |
| 主人明确说"忘了也没关系" | 0 |

### 2.3 阈值判断逻辑

```python
# 阈值判断伪代码
def classify_memory(total_score):
    if 主人明确要求记住():
        return "必要记忆"  # 直接进入必要记忆区
    
    if total_score >= 80:
        return "长期记忆"  # 立即升级
    elif total_score >= 60:
        return "待观察"    # 进入高频提及区观察
    elif total_score >= 40:
        return "临时记忆"  # 进入临时记忆区
    else:
        return "可丢弃"    # 快速过期
```

### 2.4 特殊处理规则

#### 2.4.1 主人明确指令

```
主人说"记住xxx"、"这个很重要"、"别忘了"等 → 直接标记为长期记忆
主人说"忘了也行"、"无所谓" → 降低3档存储层级
```

#### 2.4.2 上下文感知

```
主人正在处理紧急任务 → 当前任务相关记忆临时升一级
主人进入新领域 → 历史相关记忆升一级以提供参考
主人情绪低落 → 避免触发负面记忆
```

#### 2.4.3 渐进确认机制

```
当记忆价值处于临界值（55-65分）时：
1. 先标记为"待确认"
2. 观察主人后续提及情况
3. 3次有效提及 → 升级为长期记忆
4. 7天无提及 → 降级为临时记忆
```

### 3.3 自动升级/降级机制

#### 3.3.1 升级触发条件

| 触发条件 | 升级类型 | 说明 |
|:---|:---|:---|
| 主人说"记住xxx" | 即时升级 | 直接进入长期记忆区 |
| 7天内提及≥5次 | 自动升级 | 从临时区→高频区 |
| 30天内提及≥10次 | 自动升级 | 从高频区→长期区 |
| 关联簇分析发现 | 智能升级 | 批量升级关联记忆簇 |
| 主人进入新项目 | 批量升级 | 相关历史记忆升一级 |

#### 3.3.2 降级触发条件

| 触发条件 | 降级类型 | 说明 |
|:---|:---|:---|
| 主人说"忘了也行" | 即时降级 | 降1-2档 |
| 60天无提及 | 自动降级 | 高频区→临时区 |
| 30天无提及 | 自动降级 | 长期区→高频区 |
| 价值评分<30 | 标记清理 | 进入回收站，7天后删除 |
| 主人说"彻底忘掉" | 彻底删除 | 不可恢复，需确认 |

### 3.4 迟滞机制（防抖动）

```
问题：访问频率波动时，记忆在层级间频繁迁移，浪费资源
解决：Hysteresis迟滞机制

升级阈值 = 目标阈值 + 迟滞值（例如 +10分）
降级阈值 = 目标阈值 - 迟滞值（例如 -10分）

示例：
长期记忆区阈值 = 70分
升级迟滞 = +10 → 需要80分才升级
降级迟滞 = -10 → 需要60分才降级
结果：记忆在70-80分之间波动时不会频繁迁移
```

## 四、历史记录智能处理流程

### 4.1 高频内容识别算法

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          高频内容识别算法                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  1. 提及次数统计                                                            │
│     - 按记忆ID统计7天/30天/90天提及次数                                       │
│     - 区分"提及关键词"和"精确检索"                                          │
│                                                                              │
│  2. 关联度分析                                                               │
│     - 识别同时被提及的记忆簇                                                  │
│     - 如果记忆A被提及，记忆B在同一次对话中出现 → 关联+1                      │
│                                                                              │
│  3. 时序模式挖掘                                                             │
│     - 主人每周几会问某个话题 → 周期性提及预测                                 │
│     - 某个项目启动时会关联哪些记忆 → 场景触发识别                            │
│                                                                              │
│  4. 综合评分                                                                 │
│     高频分 = 精确提及×3 + 关键词提及×1 + 关联提及×0.5                        │
│                                                                              │
│  5. 升级决策                                                                 │
│     if 高频分 >= 15 and 7天内提及 >= 5:                                      │
│         触发自动升级                                                          │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 相似内容去重机制

```
┌─────────────────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  触发场景：                                                                  │
│  - 主人重复描述同一件事（可能表述不同）                                        │

│                                                                              │

│  2. 语义相似度（向量余弦）                                                    │
│     - 相似度 >= 0.80 → 标记为"可能重复"                                     │
│  3. 实体重叠度                                                                │
│     - 共同实体 >= 3个 → 标记为"可能重复"                                    │
│                                                                              │
│  处理方式：                                                                  │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  情况                           │  处理方式                          │   │
│  ├─────────────────────────────────┼────────────────────────────────────┤   │
│  │  完全重复（相似度>0.95）         │  保留最新，覆盖旧记录              │   │
│  │  高度相似（0.85-0.95）          │  合并摘要，保留版本历史            │   │
│  │  中度相似（0.70-0.85）          │  关联存储，建立引用关系            │   │
│  │  低度相似（<0.70）              │  独立存储，不处理                  │   │

│  │  ├── 触发：记忆超过7天未被精确检索                                   │   │
│  │  ├── 动作：保留核心结论，压缩过程细节                                │   │
│  │  └── 示例：完整对话记录 → 关键决策+结论                              │   │
│  │                                                                      │   │

│  │                                                                      │   │

│  │  ├── 触发：记忆超过180天无关联激活                                   │   │
│  │  ├── 动作：只保留存在性标记，可按需恢复                              │   │

│                                                                              │

│  │                                                                      │   │
│  │  Step 1: 设定相关性阈值（threshold = 0.6）                          │   │
│  │  Step 2: 筛选达标神经元进入候选池                                    │   │
│  │  Step 3: 从候选池取Top-K（候选池不足K则少激活）                      │   │
│  │  Step 4: 无达标神经元则不激活任何神经元                              │   │
│  │                                                                      │   │
│  │  def activate_neurons(query, all_neurons, K=5, threshold=0.6):     │   │
│  │      # 计算相关性分数                                               │   │
│  │      candidates = []                                                │   │
│  │      for neuron in all_neurons:                                     │   │
│  │          score = compute_relevance(query, neuron.profile)           │   │
│  │          if score >= threshold:                                     │   │
│  │              candidates.append((neuron, score))                     │   │
│  │                                                                      │   │
│  │      # 按分数排序，取Top-K                                          │   │
│  │      candidates.sort(key=lambda x: x[1], reverse=True)              │   │
│  │      activated = candidates[:K]                                     │   │
│  │                                                                      │   │
│  │      return activated  # 可能为空，也可能少于K个                    │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  效果对比                                                           │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │                                                                      │   │
│  │  场景              │ "前K个"         │ "K个相关神经元"              │   │
│  │  ─────────────────┼─────────────────┼────────────────────────────   │   │
│  │  高相关性问题多    │ 取前K个，OK     │ 取K个，OK                    │   │
│  │  高相关性问题少    │ 强行取低相关    │ 只激活达标的，精准           │   │
│  │  无相关性问题      │ 仍取前K个，浪费 │ 不激活，节省资源             │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                              │
│  核心原则：宁可少激活，不乱激活                                              │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 9.4 三大机制协同效应

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          三大机制协同效应                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  问题              │ 层级折叠       │ 智能分割       │ 稀疏激活            │
│  ─────────────────┼───────────────┼───────────────┼────────────────────    │
│  记忆越存越多      │ 按价值降级压缩 │ 事件块聚合    │ 只激活相关的        │
│  找不到想找的      │ 层级索引定位   │ 事件边界精准  │ 阈值过滤噪声        │
│  迁移太慢          │ 高层级优先     │ 完整事件块    │ 减少激活开销        │
│  恢复不完整        │ 渐进式解压     │ 事件块独立    │ 按需加载相关神经元  │
│  激活噪声大        │ -             │ -            │ 阈值+稀疏激活       │
│                                                                              │
│  协同流程：                                                                  │
│                                                                              │
│  新记忆输入 ──→ 智能分割(事件边界) ──→ 价值评估 ──→ 存储路由              │
│                                              │                               │
│                                              ▼                               │
│  查询输入 ──→ 稀疏激活(K个相关神经元) ──→ 层级折叠(按需解压) ──→ 结果     │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```
