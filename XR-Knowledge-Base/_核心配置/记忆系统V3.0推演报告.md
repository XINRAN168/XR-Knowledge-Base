### 5.2 核心风险详解

#### 🟡 R-01: L2语义校验依赖LLM API

```
触发场景：
- LLM API服务不可用
- API响应延迟过高（>500ms）
- API返回格式异常

缓解措施：
1. 接口隔离层设计
   - 定义标准接口规范
   - 支持多LLM提供商切换
2. 降级策略
   - L2不可用 → 标记待校验 → 后续补验
   - 不阻塞主流程
3. 缓存机制
   - 相同语义校验结果缓存
   - 有效期24小时

剩余风险：低（降级策略兜底）
```

#### 🟡 R-02: 校验性能开销

```
触发场景：
- 每次决策都触发四层校验
- L3外部知识库查询延迟
- 并发校验请求堆积

缓解措施：
1. 分级响应机制
   - C≥0.9999：直接执行，无感知
   - 0.99≤C<0.9999：快速+异步校验
   - C<0.99：带警告执行+二次校验
2. 异步校验
   - 低优先级校验异步执行
   - 不阻塞主流程
3. 批量校验
   - 相同类型校验批量执行
   - 减少API调用次数

剩余风险：低（分级机制确保核心流程无感知）
```
