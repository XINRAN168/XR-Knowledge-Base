### 4.2 核心风险详解

#### 🟡 R-11: 折叠信息丢失风险

```
触发场景：
- NLP摘要质量不佳，提取了错误的关键信息
- 多个细节组合才有价值，单个压缩后丢失
- 主人后来需要某个被折叠的细节

建议缓解：
1. 增加"不可折叠"标记字段
   - 主人说"这个细节很重要" → memory.folding_protected = true
2. 折叠前增加摘要预览
   - 生成摘要后展示给主人确认
3. 保留原始数据的哈希值
   - 解压时验证完整性
4. 增加"按需解压"能力
   - 高层级记忆被精确检索时自动解压
```

#### 🟡 R-12: 边界识别错误风险

```
触发场景：
- 语义边界识别错误，把同一事件分割成多个
- 把不同事件合并成一个事件块
- 漏检隐式边界（如主人换了话题但没说"对了"）

建议缓解：
1. 增加置信度阈值
   - boundary.confidence >= 0.8 才确认分割
   - 低于阈值时保留为"待确认"
2. 增加后验验证
   - 事件块形成后检测子记忆间的关联度
   - 关联度低于阈值则触发二次分割
3. 支持手动修正
   - 主人可调整事件边界
4. 记录识别准确率
   - 持续学习优化边界识别模型
```

#### 🟡 R-13: 激活不足风险

```
触发场景：
- 所有神经元相关性都低于0.6阈值
- 主人询问的问题过于宽泛
- 新领域问题缺乏相关神经元

建议缓解：
1. 阈值可配置
   - strict模式：0.7
   - normal模式：0.6
   - loose模式：0.5
2. 降级策略
   - 低于阈值时，自动降低到0.5重新筛选
   - 仍不足则激活默认神经元（如"谋士层"）
3. 增加"探索模式"
   - 主人明确说"随便聊聊"时，降低阈值
```

#### 🟡 R-14: 三机制复杂度风险

```
触发场景：
- 三机制相互依赖，单个测试通过但集成后出问题
- 出现问题时难以定位是哪个机制的责任
- 维护成本高

建议缓解：
1. 分层设计
   - 分割层：独立处理输入，不涉及折叠和激活
   - 折叠层：独立处理压缩，不依赖分割结果
   - 激活层：独立处理激活，使用各层输出
2. 独立测试
   - 每个机制单独测试覆盖率≥90%
   - 集成测试验证接口正确性
3. 日志分层
   - 分割日志、折叠日志、激活日志分开记录
4. 逐步上线
   - 先上线分割，验证稳定后再开折叠
```
